# AGENTS.md

## Project Mission

Keep documentation accurate, skimmable, and useful to builders. Prefer verified facts, examples that run, and clear labels for experimental content.

## How To Work

- Verify fast-moving facts before updating.
- Prefer official sources.
- Label community reports clearly.
- Do not copy long passages from external sources.
- Keep examples short and complete.
- Remove stale content instead of adding contradictory notes.

## Structure

- `README.md` is the front page.
- `docs/` is for long-form explanations.
- `examples/` is for prompts and workflows.
- `templates/` is for copy-paste files.

## Style

- Lead with the practical answer.
- Use tables for resource lists and comparisons.
- Add dates for launch timing, pricing, availability, and version-specific claims.
- Avoid overstated words unless backed by evidence.

## Verification

Before finishing:

- Check Markdown rendering.
- Check links you touched.
- Confirm headings still match the table of contents.

## Safety

- Do not include private conversations, screenshots, tokens, or user data.
- Do not imply unofficial projects are endorsed by xAI.
